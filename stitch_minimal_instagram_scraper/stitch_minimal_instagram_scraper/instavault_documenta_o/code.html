Technical Documentation

A robust, developer-first framework for high-scale data extraction. This guide covers the low-level handshake protocols and the orchestration layer of the InstaVault CLI.

Autenticação Local (Sessão)

A autenticação no InstaVault utiliza um sistema de persistência de sessão local via arquivos .iv_session. Uma vez autenticado, o token de portador (Bearer Token) é armazenado de forma criptografada no diretório home do usuário, permitindo operações CLI subsequentes sem re-entrada de credenciais.

Arquitetura de Extração

O InstaVault opera em um modelo híbrido. A interface de orquestração e gerenciamento de filas é construída em **Node.js**, enquanto o núcleo de processamento intensivo e parsing de DOM é delegado para subprocessos **Python**. Esta abordagem garante a escalabilidade do I/O assíncrono e a precisão das bibliotecas de extração de dados.

IPC BRIDGE

Comunicação via Standard Streams (STDIN/STDOUT) em formato JSON.

javascript worker_spawner.js
NODE.JS v18+ 
const { spawn } = require('child_process');

// Initialize the Python scraping core
const pythonProcess = spawn('python3', ['./core/extractor.py']);

pythonProcess.stdout.on('data', (data) => {
    const result = JSON.parse(data.toString());
    processMetricQueue(result.payload);
});

pythonProcess.stderr.on('data', (err) => {
    console.error(`[WORKER_ERROR]: ${err}`);
});

// Sending instructions to Python via STDIN
pythonProcess.stdin.write(JSON.stringify({ 
    task: 'EXTRACT_POSTS', 
    target: '@tech_insider' 
}));
